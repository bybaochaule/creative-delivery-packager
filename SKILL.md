# Creative Delivery Packager

Use this skill to turn creative project files into clean, professional, buyer-friendly, client-ready, and marketplace-ready delivery packages.

## Core Behavior

Create clear delivery systems with:

- Folder structure
- File format map
- Naming convention
- Export settings
- Preview rules
- Thumbnail rules
- ZIP packaging plan
- Source-file separation
- Final-file separation
- README or instruction file
- License placement
- Version control
- Quality-control checklist
- Delivery checklist
- Client or customer handoff note

## Operating Rules

Always:

1. Identify the product type and final user.
2. Separate editable source files from exported final files.
3. Separate preview images from usable final assets.
4. Use predictable folder names.
5. Use consistent file names.
6. Explain the purpose of each file format.
7. Include export quality settings when relevant.
8. Include compatibility notes when relevant.
9. Include README and license files when relevant.
10. Make previews accurately represent included files.
11. Remove drafts, duplicates, temporary files, and private files.
12. Test the final ZIP package before delivery.

Never help with:

- Malware or hidden scripts
- Deceptive ZIP files
- Fake licenses
- Stolen assets
- Unlicensed fonts, mockups, templates, audio, or video
- Unauthorized trademarks, logos, or copyrighted characters
- Misleading previews
- Private client files without permission
- Claims of marketplace approval, legal clearance, or guaranteed sales

## Workflow

### 1. Identify Context

Determine:

- Product name
- Product type
- Target buyer or client
- Marketplace or delivery platform
- Required file formats
- Source files to include or exclude
- Final exports needed
- Preview and thumbnail needs
- Software used
- License requirements
- Versioning needs
- Delivery size limits

If information is missing, make professional assumptions and label them clearly.

### 2. Choose Output Mode

Use the most relevant mode:

- Full delivery system
- Marketplace product package
- Client handoff package
- Export settings plan
- Naming convention
- Preview and thumbnail plan
- ZIP packaging workflow
- Automation workflow
- Delivery audit

### 3. Create File Format Map

Use this structure:

| Format | Purpose | Export settings | Destination | Quality check |
|---|---|---|---|---|
| PNG | Transparent assets | Transparent background, high resolution | PNG folder | Check edges and transparency |
| JPG | Previews or photos | High quality, flattened background | Preview folder | Check compression and readability |
| SVG | Vector or cut files | Clean paths, scalable vectors | SVG folder | Open in vector software |
| PDF | Printables or guides | Correct page size, margins, fonts | PDF folder | Open and test links |
| PSD/AI | Editable source files | Organized layers and groups | Source folder | Open and inspect |
| MP4 | Video deliverables | Correct resolution, bitrate, audio | Video folder | Play full file |
| ZIP | Final delivery | Compressed package | Upload package | Extract and test |

### 4. Create Folder Structure

Use only folders the project needs.

Default structure:

```text
Product_Name_Delivery/
  00_READ_ME_FIRST/
  01_Final_Files/
  02_Print_Ready/
  03_Web_Ready/
  04_Editable_Source_Files/
  05_Previews/
  06_Thumbnails/
  99_Archive/
```

Marketplace structure:

```text
Product_Name_V1/
  00_READ_ME_FIRST/
  01_Final_PNG/
  02_Final_SVG/
  03_Print_Ready_PDF/
  04_Preview_Images/
  05_License/
```

Client handoff structure:

```text
Client_Project_Final_Delivery/
  00_READ_ME_FIRST/
  01_Final_Exports/
  02_Print_Files/
  03_Web_Files/
  04_Source_Files/
  05_Brand_Guide/
  06_License_And_Usage/
```

### 5. Create Naming Convention

Recommended formula:

```text
productname_assettype_descriptor_size_version.format
```

Examples:

```text
botanical_clipart_rose_3000px_v1.png
wedding_invitation_template_5x7_v1.pdf
logo_primary_color_rgb_v1.png
logo_primary_black_v1.svg
video_intro_1920x1080_30fps_v1.mp4
```

Avoid:

- `final.png`
- `final_final.png`
- `copy.png`
- `test.svg`
- `untitled-1.psd`
- Random numbers without meaning

### 6. Create Preview Rules

Preview images should:

- Show only included files
- Clearly show file quantity and formats
- Be readable on mobile
- Use licensed mockups only
- Label mockups as display-only if not included
- Avoid fake software compatibility claims
- Avoid copyrighted logos or brand assets unless authorized

### 7. Create README Contents

Include:

- Product name
- What is included
- File formats
- How to use the files
- Software compatibility
- Printing or editing notes
- License summary
- Troubleshooting
- Support note
- Version and date

### 8. Create ZIP Packaging Checklist

Before final ZIP:

- Remove drafts and test files
- Remove duplicates
- Remove hidden or private files
- Confirm all promised formats are included
- Confirm README and license files are included
- Open representative files
- Compress the final folder
- Extract the ZIP into a clean test folder
- Open files from the extracted copy
- Confirm upload size limits

ZIP filename formula:

```text
productname_delivery_v1_yyyy-mm-dd.zip
```

## Final Response Format

When responding, provide:

1. Assumptions
2. Delivery strategy
3. File format map
4. Folder structure
5. Naming convention
6. Preview and thumbnail plan
7. README contents
8. ZIP packaging checklist
9. Quality-control checklist
10. Handoff note or marketplace note
